Boilerplate code:

Visit our github repository to get up and running with a Node.js boilerplate. This is an open-source project and we welcome you to submit any and all comments, questions, and pull requests.

If you want to get up an running with a LEAN, barebones, single-page web-application with client-side mining and server-side mining verification, clone the repository and follow the install instructions below. Cloning this boilerplate, following the instructions below, and running the application produces a copy of this website. Following our installation guide requires that you have homebrew installed:

1. Installing Node.js and NPM:
brew update brew install node sudo npm install npm -g
2. Installing dependencies:
npm install
3. Run the Node.js application:
npm run dev
4. Visit the locally running application:
http://localhost:3000

API:

This repository also has a standalone client-side mining controller file in /dist/ClientMiner.js. This script provides a constructor function for a javascript client-side miner that you can instantiate with:

var miner = new ClientMiner();

You can include the standalone script in a project and begin using the methods on the ClientMiner() constructor right out of the box, but you must include the following script before including the ClientMiner.js file:

<script src="https://coinhive.com/lib/coinhive.min.js"></script>

The ClientMiner.js controller extends the Monero Mining interface provided by CoinHive by inheriting several methods and reorganizing them for custom control flow. Below is our documentation for each of the methods on the ClientMiner constructor:

miner.getUser()

() => String
If the miner is set up, returns the unique string identifier associated with the current User.

miner.getStatistics()

() => Object
Returns the an object with the current hashes per second, total hashes and accepted hashes associated with a user at the time when the method is called.

miner.startMining()

() => ()
Begins the mining session. miner.setMiner() must be called before the method else an error is thrown.

miner.setMiner()

() => () Instantiates a new user so that a new mining session can begin.

miner.pauseMiner()

() => ()
Pauses miner. Calling this method does not change the unique identifier associated with the miner.

miner.isRunning()

() => Boolean
Returns true or false depending on whether the miner is currently running.

miner.getNumThreads()

() => Integer
Returns the number of threads currently being used for mining.

miner.setNumThreads(threads)

Integer => ()
Sets the number of threads currently being used for mining. The number argument threads must be an integer between 0 and 8, inclusive.
